## 版本控制方案

Tiny-LLM 遵循[语义化版本](https://semver.org/)：

- **主版本号**：不兼容的 API 变更
- **次版本号**：向后兼容的新功能
- **修订号**：向后兼容的错误修复

## 发布渠道
