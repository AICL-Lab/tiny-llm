# v2.0.0

发布日期: 2024-01-01

我们很高兴地宣布 Tiny-LLM v2.0.0，这是一个支持 INT8 量化的主要版本！

## 主要功能

### INT8 量化 (W8A16)

v2.0.0 的核心功能是全面支持 INT8 权重量化：

- **内存减少 50%** 用于模型权重
- **推理速度提升 2 倍** 在内存受限的工作负载上
- **精度损失极小** 采用分组量化

```cpp
// 加载量化模型
InferenceEngine engine;
engine.load("model-int8.bin");

// 正常生成
engine.generate("你好，世界！");
```

### 新架构

使用现代 C++17 设计完全重写的核心：

- `Result\<T\>` 实现无异常错误处理
- `KVCacheManager` 实现高效生成
- 模块化的 `TransformerLayer` 架构

### CUDA 优化

- 融合注意力内核
- 优化的 INT8 矩阵乘法
- 改进的内存合并

## 破坏性变更

### API 变更

API 已为 v2.0.0 重新设计：

| v1.x | v2.0.0 |
|------|--------|
| `loadModel()` | `load()` |
| `generateText()` | `generate()` |
| `getError()` | `result.error()` |

### 配置变更

配置现在使用结构体：

```cpp
// 旧版 (v1.x)
engine.set("max_tokens", 100);

// 新版 (v2.0.0)
GenerationConfig config;
config.max_tokens = 100;
engine.generate(prompt, config);
```

### 模型格式

v2.0.0 使用新的模型格式。转换您的模型：

```bash
tinyllm-convert --input model-v1.bin --output model-v2.bin
```

## 新组件

### Result\<T\>

无异常错误传播：

```cpp
Result<std::string> result = engine.generate("你好");
if (!result.ok()) {
    std::cerr << result.error_message() << std::endl;
}
```

### KVCacheManager

高效的键值缓存：

```cpp
KVCacheManager cache;
cache.initialize(model_config, cache_config);
auto seq_id = cache.create_sequence();
```

## 性能

### 基准测试 (RTX 4090 上的 LLaMA-7B)

| 指标 | v1.x | v2.0.0 | 提升 |
|------|------|--------|------|
| 内存 (INT8) | 13 GB | 6.5 GB | 50% |
| 预填充 | 650 tok/s | 850 tok/s | 31% |
| 解码 | 48 tok/s | 65 tok/s | 35% |

## 迁移指南

### 步骤 1：更新模型文件

```bash
tinyllm-convert --input old-model.bin --output new-model.bin
```

### 步骤 2：更新代码

```cpp
// 旧版
TinyLLM engine;
engine.loadModel("model.bin");
auto text = engine.generateText("你好");

// 新版
InferenceEngine engine;
engine.load("model.bin");
auto result = engine.generate("你好");
if (result.ok()) {
    std::cout << result.value() << std::endl;
}
```

### 步骤 3：更新构建

```bash
# v2.0.0 需要 C++17
cmake -S . -B build -DCMAKE_CXX_STANDARD=17
cmake --build build -j$(nproc)
```

## 已知问题

- INT4 量化尚不支持
- Flash Attention 需要 CUDA 11.6+
- Windows 支持处于实验阶段
