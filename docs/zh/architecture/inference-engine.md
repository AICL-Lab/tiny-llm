# 架构概览

Tiny-LLM 在设计上注重简洁性和教育价值，同时保持生产级的性能。

## 核心组件

```
┌─────────────────────────────────────────────────────┐
│                  InferenceEngine                     │
│                    (公共 API)                        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │
│  │ ModelConfig │  │   Result\<T\> │  │  KVCache    │ │
│  │             │  │             │  │  Manager    │ │
│  └─────────────┘  └─────────────┘  └─────────────┘ │
│                                                      │
├─────────────────────────────────────────────────────┤
│               TransformerLayer                       │
│  ┌─────────────────┐    ┌─────────────────┐        │
│  │  Attention      │    │      FFN        │        │
│  │  (W8A16)        │    │    (W8A16)      │        │
│  └─────────────────┘    └─────────────────┘        │
├─────────────────────────────────────────────────────┤
│               QuantizedWeight                        │
│         (INT8 权重 + 每组缩放因子)                   │
├─────────────────────────────────────────────────────┤
│                  CUDA 内核                           │
│   (注意力、前馈网络、量化、内存操作)                  │
└─────────────────────────────────────────────────────┘
```

## 组件职责

### InferenceEngine

协调模型加载和生成的公共 API：

- `load()` - 从磁盘加载模型权重
- `generate()` - 从提示词生成文本
- `encode()` - 对输入文本进行分词

### Result\<T\>

无异常错误传播机制：

```cpp
Result<void> result = engine.load("model.bin");
if (!result.ok()) {
    // 处理错误
    std::cerr << result.error() << std::endl;
}
```

### ModelConfig

保存模型超参数：

| 参数 | 类型 | 描述 |
|------|------|------|
| `vocab_size` | `size_t` | 词汇表大小 |
| `hidden_dim` | `size_t` | 隐藏层维度 |
| `num_layers` | `size_t` | Transformer 层数 |
| `num_heads` | `size_t` | 注意力头数 |
| `max_seq_len` | `size_t` | 最大序列长度 |

### QuantizedWeight

带每组缩放因子的 INT8 量化权重：

- 权重以 INT8 存储
- 每组独立的缩放因子用于反量化
- 支持 W8A16 推理模式

### TransformerLayer

单个 Transformer 层包含：

- 多头自注意力
- 前馈网络
- 层归一化
- 残差连接

### KVCacheManager

管理键值缓存以实现高效生成：

- 预分配的缓存槽位
- 每序列缓存管理
- 内存高效的缓存策略

## 内存布局

```
┌────────────────────────────────────────┐
│           权重内存                      │
│  ┌──────────────────────────────────┐  │
│  │   嵌入表 (FP16)                  │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│   层权重 (INT8 + 缩放因子)            │  │
│  │   ┌────────────────────────────┐ │  │
│  │   │  Q, K, V 投影              │ │  │
│  │   │  输出投影                  │ │  │
│  │   │  FFN 上下/下投影           │ │  │
│  │   └────────────────────────────┘ │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│  │   输出层 (FP16)                 │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

## 数据流

```
输入文本
    │
    ▼
┌─────────────┐
│   分词器    │
└─────────────┘
    │
    ▼
Token IDs
    │
    ▼
┌─────────────────┐
│    嵌入层       │  ← FP16 查找
└─────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         Transformer 层              │
│  ┌─────────────────────────────┐    │
│  │     自注意力                 │    │  ← W8A16 量化
│  │  (Q×K^T / √d) → Softmax → V │    │
│  └─────────────────────────────┘    │
│              │                      │
│              ▼                      │
│  ┌─────────────────────────────┐    │
│  │     前馈网络                 │    │  ← W8A16 量化
│  │     Linear → GELU → Linear  │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────┐
│    输出层        │  ← FP16 Logits
└─────────────────┘
    │
    ▼
┌─────────────────┐
│    采样         │
└─────────────────┘
    │
    ▼
输出 Tokens
```
